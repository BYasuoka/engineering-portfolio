Budgeting and Tax Planning Scripts

This project is a personal finance scripting toolkit built in Python. Across the files in the folder, the work evolves from quick back-of-the-envelope calculations into CSV-driven tax analysis, retirement scenario modeling, and structured budgeting outputs shown with Matplotlib tables.

Back to all project portfolios

Project Scope

Decision support for real budgeting questions

The scripts answer practical questions like where take-home pay goes after federal and state taxes, how retirement contributions affect spendable income, and how commuting changes monthly cost.

Inputs

CSV tax brackets plus user-entered assumptions

Federal and state bracket files are stored as CSVs, while the user provides gross income, retirement contribution levels, commute distance, or state selection depending on the specific script.

What The Folder Shows

A project that matured through iteration

Early comparison workflow

`Tax_calculator.py` compares California and Colorado by computing progressive federal and state taxes, then visualizes different monthly budget splits for each location.

Interactive scenario modeling

`Tax_calculator_2.py` expands the idea by letting the user choose a state and model how 401(k) and Roth IRA contributions change monthly net income.

Pay-period-specific refinement

The Wisconsin-focused scripts (`Tax_calculator_MKE.py` and `Tax_calculator_MKE_monthly.py`) adapt the same logic for biweekly and monthly planning, showing a nice awareness that budgeting depends on how pay is actually received.

Supporting utilities

`DrivingExpense_Caclulator.py` estimates commute fuel cost, while `401kMatch.py` is a fast calculator for employer match comparisons. Together they show a toolkit mindset rather than a single one-off script.

Technical Approach

Data-driven progressive tax calculation

The strongest architectural choice in these scripts is storing tax brackets in CSV files with `min`, `max`, and `rate` columns. That keeps the computational logic reusable while making state-specific updates a data maintenance task instead of a code rewrite.

  • Bracket rows are loaded and normalized, including support for an infinite upper bound.
  • Taxes are computed progressively by iterating bracket intervals and taxing only the relevant slice of income.
  • Pretax deductions are handled separately so retirement scenarios can be modeled against adjusted taxable income.

Presentation

Structured financial outputs instead of raw console text

Rather than stopping at a printed number, the later scripts build scenario tables and budget allocation tables with Matplotlib. That makes the output more readable and closer to a decision-support dashboard.

  • Scenario tables compare no retirement, 401(k)-only, Roth-only, and combined plans.
  • Budget splits translate take-home pay into needs, wants, and savings targets.
  • Monthly and biweekly variants adapt the same logic to different pay cadences.

Why This Project Is Strong

It connects coding directly to decision making

A lot of small automation projects stop at “the script runs.” This set of files goes a step further by turning tax brackets, contribution choices, and recurring expenses into clear tradeoff views. That is a useful engineering instinct: build tools that help people evaluate options instead of only producing intermediate calculations.
